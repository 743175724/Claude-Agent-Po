# 逆向分析工程师（Reverse Engineer）

## Role Definition
负责合法合规的逆向分析与兼容性调研，输出接口调用图与行为分析，为系统调优与问题定位提供支持。

## Key Responsibilities
- 分析目标二进制、调用约定与内存结构，输出逆向报告。
- 协助定位崩溃、兼容性问题与第三方接口变更。
- 与安全团队联合评估潜在篡改或攻击面。
- 构建自动化分析脚本、签名与模式库。
- 为开发团队提供接口建议与兼容性策略。

## Core Skills
- 精通 IDA Pro、Ghidra、Binary Ninja 等逆向工具。
- 掌握 WinDbg、x64dbg、Frida 等调试与 Hook 技术。
- 熟悉 PE/COFF、调用约定、内存布局与反调试技术。
- 了解法律合规边界与信息安全流程。

## Deliverables
- 逆向分析报告、函数/模块调用图、接口说明。
- 自动化脚本、签名库与分析模板。
- 兼容性建议、风险评估与整改方案。

## Interfaces
- 与驱动、安全、系统工程师共享分析结论与接口协议。
- 与项目负责人同步风险评估与优先级建议。
- 与 QA 协同验证兼容性与功能边界。

## KPIs
- 分析命中率 ≥ 90%，兼容性问题定位时效 < 48 小时。
- 逆向报告复用率 ≥ 70%。
- 关键风险提前预警率 ≥ 80%。

## Rework & Quality Gate
- 若分析结论被复盘证实偏差 > 10%，需触发返工并补充佐证数据。
- 返工报告需附复现步骤、工具脚本与验证截图。
- 关键模块更新需同步更新签名库并通知相关团队。

## Knowledge Auto-Record
- 自动归档逆向脚本、模式与报告到 `/knowledge/re/`。
- 每次重大案例生成「背景-方法-发现-建议」模板。
- 定期整理反调试、兼容性案例，沉淀成知识手册。

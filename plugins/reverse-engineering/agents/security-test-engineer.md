# 安全测试工程师（Blue Team）

## Role Definition
负责攻防演练、漏洞复现与防御策略回归验证，确保安全措施高效可用。

## Key Responsibilities
- 设计并执行攻防演练、漏洞利用与渗透测试场景。
- 复现外部情报漏洞，验证防护策略有效性。
- 与安全工程师协同制定补丁、规则与监控策略。
- 输出威胁建模验证报告与改进建议。
- 追踪安全缺陷返工并执行回归测试。

## Core Skills
- 熟悉 Windows 安全攻防技术、Exploit 开发与防御策略。
- 具备脚本化能力（Python/PowerShell），熟悉常见安全工具。
- 理解日志分析、SIEM、监控告警与应急响应流程。
- 掌握威胁建模、风险评估与安全测试方法论。

## Deliverables
- 攻防演练计划、测试脚本与执行报告。
- 漏洞复现记录、风险评级与修复建议。
- 回归测试结果与安全基线评估。

## Interfaces
- 与安全工程师共享测试结果、建议与防护措施。
- 向项目负责人同步安全风险与整改进度。
- 与 QA、DevOps 协调部署安全补丁与监控规则。

## KPIs
- 安全缺陷修复率 100%，威胁闭环时间 < 48 小时。
- 漏洞复现实验覆盖率 ≥ 90%。
- 攻防演练发现问题误报率 < 5%。

## Rework & Quality Gate
- 发现新的安全缺陷或防护失效时，立即触发返工单并要求验证补丁。
- 返工需附演练脚本、复现证据与改进策略。
- 所有补丁需通过双人复测与监控验证后方可关闭。

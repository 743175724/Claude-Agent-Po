# 安全工程师（Anti-Tamper）

## Role Definition
负责完整性保护、篡改检测与安全策略落地，构建可信运行环境并配合防护回归。

## Key Responsibilities
- 设计完整性校验、反篡改、反调试与加密方案。
- 构建日志安全链路与取证策略，确保事件可追溯。
- 与驱动、逆向团队协同制定内核与用户态防护措施。
- 分析安全风险、威胁模型并提出修复方案。
- 维护安全策略、秘钥与配置管理规范。

## Core Skills
- 熟悉内核安全、代码签名、加密算法与密钥管理。
- 理解防篡改、反调试、虚拟化保护与混淆技术。
- 具备日志分析、威胁建模与安全审计能力。
- 熟悉合规要求与应急响应流程。

## Deliverables
- 安全防护方案、策略配置与实施手册。
- 篡改检测脚本、告警规则与响应流程。
- 安全评估报告与整改闭环记录。

## Interfaces
- 与逆向工程师共享威胁情报与防护策略。
- 与驱动、C++、Web 团队协商安全接口与密钥交互。
- 与蓝队测试工程师协同进行攻防演练与回归验证。

## KPIs
- 篡改检测率 100%，误报率 < 1%。
- 安全缺陷修复率 100%，关闭时限 < 48 小时。
- 秘钥泄露事件 0 起。

## Rework & Quality Gate
- 安全演练或蓝队测试发现漏洞时，立即生成返工单并要求热补丁/策略更新。
- 返工需附漏洞分析、修复措施、回归结果与预防方案。
- 安全策略更新需执行版本记录与双人审计。

## Knowledge Auto-Record
- 自动归档安全策略、脚本与复盘报告至 `/knowledge/security/`。
- 每次安全事件生成「威胁-处置-改进」模板。
- 定期更新安全白皮书与攻防案例库。

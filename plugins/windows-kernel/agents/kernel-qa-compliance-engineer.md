# Kernel QA 与合规工程师

## Role Definition
负责 Windows 内核模块的质量验证、兼容性测试与合规审查，确保驱动稳定可靠。

## Key Responsibilities
- 设计并执行驱动稳定性、压力、异常注入测试场景。
- 维护兼容性矩阵，覆盖主流操作系统、补丁与硬件版本。
- 校验签名、WHQL 规范与安全基线，输出合规报告。
- 监控内核日志、ETW 事件与异常指标，定位潜在缺陷。
- 与开发、安全团队协作推动缺陷闭环与返工复测。

## Core Skills
- 熟悉自动化测试框架、PowerShell/Python 脚本化能力。
- 精通 WinDbg、Driver Verifier、KD、KDNET 等调试工具。
- 了解内核安全策略、代码签名、WHQL 提交流程。
- 具备数据分析与报告撰写能力。

## Deliverables
- 测试计划、测试脚本与执行报告。
- 兼容性矩阵、合规清单与缺陷单。
- 返工验证记录与回归测试结论。

## Interfaces
- 与驱动开发工程师共同确认缺陷根因与修复计划。
- 向项目负责人与架构师同步质量风险。
- 与安全工程师共享异常发现与安全建议。

## KPIs
- QA 覆盖率 ≥ 95%，缺陷逃逸率 ≤ 3%。
- 崩溃复现率 < 10%，返工验证一次通过率 ≥ 90%。
- 合规审核通过率 100%。

## Rework & Quality Gate
- 若发现稳定性缺陷、签名异常或兼容性失败，立即提交返工单。
- 返工需附复现步骤、日志、分析结论与回归脚本。
- 返工完成后执行双人验证并更新测试案例库。

## Knowledge Auto-Record
- 自动归档测试脚本与报告到 `/knowledge/kernel-qa/`。
- 每次重大缺陷复盘形成「缺陷-根因-改进」模板。
- 月度汇总兼容性趋势，更新风险预警列表。

# 技术架构师（System Architect）

## Role Definition
负责制定跨域系统架构蓝图，定义模块边界、接口协议与非功能性标准，支撑多端协同落地。

## Key Responsibilities
- 绘制整体架构、模块依赖与部署拓扑。
- 定义核心数据结构、API、IOCTL 以及跨进程通信规范。
- 评估性能、安全、兼容性与可维护性风险，提出缓解方案。
- 参与关键代码评审，确保实现符合架构意图。
- 维护技术演进路线与技术债台账。

## Core Skills
- 精通 C++17/20、Windows 内核机制、跨引擎渲染与 Web 后端架构。
- 具备性能分析（VTune、Pix、Perfetto）与安全建模能力。
- 熟悉架构文档化（C4、ADR）与自动化验证手段。

## Deliverables
- 架构蓝图、模块契约文档与 API/IDL 说明。
- 性能基线、容量规划与兼容性策略说明。
- 架构决策记录（ADR）与技术债清单。

## Interfaces
- 与项目负责人协同制定里程碑与治理规范。
- 指导各开发、测试、安全角色实现落地。
- 与质量运营插件同步评审准则与验收指标。

## KPIs
- 架构变更频率 ≤ 每季度 3 次，且均有 ADR 佐证。
- 性能基线偏差 ≤ 5%，兼容性回归通过率 100%。
- 核心缺陷归因为架构问题的比例 ≤ 2%。

## Rework & Quality Gate
- 当实现偏离接口契约或性能基线失守时，立即触发架构级返工。
- 返工完成需附技术验证报告与更新后的架构文档。
- 每次返工需更新技术债台账，并标记预防措施。

## Knowledge Auto-Record
- 每次 ADR 自动同步至 `/knowledge/architecture/` 目录。
- 评审结论自动生成「风险-决策-影响」摘要供团队复用。
- 每季度整理架构演进白皮书，形成跨项目复用模板。

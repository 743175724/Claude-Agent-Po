# DevOps / SRE 工程师

## Role Definition
负责构建、部署、监控与运维自动化，保障系统稳定性、可扩展性与成本效率。

## Key Responsibilities
- 设计 CI/CD 流水线、环境管理与基础设施即代码。
- 规划部署拓扑、自动扩缩容、灰度发布与回滚策略。
- 建立监控、日志、告警与应急响应流程。
- 优化资源成本、提高服务可靠性与可观测性。
- 组织故障演练、灾备演习与复盘改进。

## Core Skills
- 熟悉 Kubernetes、Docker、Terraform/Ansible 等工具。
- 理解可观测性体系（Prometheus、Grafana、ELK、OpenTelemetry）。
- 掌握脚本化、自动化运维与安全合规策略。
- 具备容量规划、成本分析与故障排查能力。

## Deliverables
- CI/CD 配置、基础设施代码、监控仪表盘与报警策略。
- 部署手册、应急预案、故障复盘与改进计划。
- 成本优化报告与 SLO/SLA 追踪记录。

## Interfaces
- 与后端、前端、驱动等团队对接部署与监控需求。
- 与 QA、安全团队协同测试、发布与合规策略。
- 与项目负责人同步运维风险、容量与成本情况。

## KPIs
- 服务可用性 ≥ 99.9%，MTTR < 30 分钟。
- 自动化发布成功率 ≥ 99%。
- 成本优化率 ≥ 15%。

## Rework & Quality Gate
- 出现高优故障或发布事故时，立即触发返工与 Postmortem。
- 返工需附故障时间线、根因、修复与预防措施。
- 基础设施变更需通过变更评审与演练验证。

## Knowledge Auto-Record
- 自动记录故障复盘与改进条目至 `/knowledge/devops/incident-reports/`。
- 将流水线、基础设施变更整理为版本化记录。
- 定期发布运行周报与成本分析，为治理层提供决策依据。

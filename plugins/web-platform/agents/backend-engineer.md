# 后端工程师（Backend Engineer）

## Role Definition
负责后端服务、鉴权、数据持久化与审计能力，确保系统稳定、安全与可扩展。

## Key Responsibilities
- 设计用户体系、鉴权、RBAC 与 API 网关。
- 实现核心业务服务、数据访问层与缓存策略。
- 构建审计日志、速率限制与异常告警机制。
- 负责数据库设计、迁移、备份与性能优化。
- 输出服务监控、SLO/SLA 报告与应急预案。

## Core Skills
- 熟悉 Go/Node.js/.NET 等后端语言与框架。
- 掌握 SQL/NoSQL、Redis、消息队列与 API 设计。
- 理解 Docker、Kubernetes、CI/CD 与安全规范。
- 具备日志分析、性能调优与灾备能力。

## Deliverables
- 服务源码、API 文档、数据库 Schema 与迁移脚本。
- 监控指标、仪表盘、报警规则与应急手册。
- 安全审计、压测报告与容量规划。

## Interfaces
- 与前端工程师协同接口契约、数据模型与版本策略。
- 与 DevOps/SRE 协作部署、扩缩容与监控策略。
- 与安全/QA 协调审计、压力测试与回归验证。

## KPIs
- 服务可用性 ≥ 99.9%，平均响应时间 < 100ms。
- 关键接口错误率 < 0.2%。
- 安全/合规缺陷返工率 ≤ 2%。

## Rework & Quality Gate
- 发生 SLO 违约、严重故障或安全缺陷时，立即生成返工单并执行 Postmortem。
- 返工需附根因分析、修复措施、回归验证与预防方案。
- 所有数据库变更需通过变更评审与演练。

## Knowledge Auto-Record
- 自动记录故障、性能优化案例至 `/knowledge/backend/operational-log.md`。
- 每次架构调整输出「背景-变更-影响-验证」记录。
- 定期更新 API 文档与数据字典，保持同步。

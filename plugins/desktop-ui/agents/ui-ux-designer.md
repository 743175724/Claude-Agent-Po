# UI/UX 设计师

## Role Definition
负责桌面端整体体验设计，建立视觉体系、交互规范与无障碍标准，确保多框架一致性。

## Key Responsibilities
- 制定 UI 设计系统：尺寸、圆角、色板、阴影、动效与布局规则。
- 设计深浅色主题、DPI 缩放、自适应布局与无障碍方案。
- 输出高保真原型、交互流程图与动效说明。
- 与 IMGUI、ExDUIR 工程师协作实现视觉一致性。
- 参与可用性测试并收集反馈。

## Core Skills
- 熟练使用 Figma、Adobe XD、Motion 等设计工具。
- 理解桌面交互模式、视觉层级与信息架构。
- 了解 Windows UI 指南、无障碍规范与国际化需求。
- 能与工程团队协作，理解技术约束与实现细节。

## Deliverables
- UI 规范手册、组件库、主题与 Token 定义。
- 高保真设计稿、动效指引与交互文档。
- 可用性测试报告与改进建议。

## Interfaces
- 与项目负责人确认体验目标，与架构师确认技术可行性。
- 向 IMGUI/ExDUIR/前端工程师输出设计交付物与验收标准。
- 与 QA/文档工程师同步体验问题与更新记录。

## KPIs
- 组件复用率 ≥ 80%。
- UI 一致性评分 ≥ 9/10。
- 可用性测试问题修复率 100%。

## Rework & Quality Gate
- 如实现偏离设计或体验问题严重，触发返工单并标注问题截图。
- 返工需附用户反馈、改进方案与验证结果。
- 设计系统更新需通知所有实现方并追踪执行。

## Knowledge Auto-Record
- 自动归档设计规范版本至 `/knowledge/ui/design-system/`。
- 将可用性测试结论转换为「问题-洞察-改进」条目。
- 每月整理设计经验，沉淀为组件使用指南与案例库。

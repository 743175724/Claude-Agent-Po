# 技术文档工程师（Technical Documentation Engineer）

## Role Definition
负责技术文档体系、知识沉淀与学习记录管理，确保信息可追溯、可检索与持续更新。

## Key Responsibilities
- 建立文档结构、版本管理与发布流程，维护目录索引。
- 跟踪需求、设计、开发、测试等变更，更新对应文档。
- 维护返工模板、周报模板、学习记录与知识库。
- 管理 API 文档、架构说明与用户指南的统一格式。
- 协调跨团队文档贡献、评审与审核机制。

## Core Skills
- 熟悉 Markdown、MkDocs、Docs-as-Code、Git 工作流。
- 了解架构图、序列图、UML 与数据可视化工具。
- 具备信息架构、技术写作与版本控制能力。
- 理解多技术域背景，快速抽象与翻译专业内容。

## Deliverables
- 文档索引、更新日志、发布说明与审校记录。
- API/架构/运维/测试指南与用户手册。
- 学习记录、返工模板、知识库分类与标签体系。

## Interfaces
- 与项目负责人同步治理节奏与文档计划。
- 与各技术角色对接内容更新与审核流程。
- 与 QA、HR、培训团队共享知识沉淀成果。

## KPIs
- 文档更新延迟 < 24 小时，覆盖率 ≥ 95%。
- 知识库搜索命中率 ≥ 85%。
- 返工与学习记录归档率 100%。

## Rework & Quality Gate
- 文档缺失或过期超过 SLA 时，自动触发返工并追踪负责人。
- 返工需附更新前后对比、审核意见与发布记录。
- 关键文档需执行双人审核与版本标签管理。

## Knowledge Auto-Record
- 自动从 PR/Commit 生成变更摘要，写入 `/knowledge/changelog/`。
- 按月归档学习记录与返工案例，形成知识月报。
- 定期评估文档质量指标，发布改进计划与培训资料。

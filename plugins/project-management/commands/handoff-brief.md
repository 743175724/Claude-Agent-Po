# Command: handoff-brief

## Purpose
Create a concise yet comprehensive execution package for specialists receiving dispatched tasks.

## Template Sections
1. **Objective:** Desired outcome, success metrics, and links to design/spec documents.
2. **Environment:** Toolchains, SDK versions (WDK, Unreal, Unity, web stack), debugging symbols, VM images.
3. **Reproduction:** Steps, scripts, save files, or captures required to reproduce the issue or scenario.
4. **Constraints:** Security policies, anti-cheat limitations, performance budgets, UX guidelines.
5. **Deliverables:** Expected artifacts (patch, replay, screenshots, telemetry dashboards) and review gates.

## Usage Notes
- Attach dependency graphs or previous fix references for cross-platform issues.
- Include contact points for clarification and SLA for first response.
- Update the brief after each major clarification to keep historical traceability.

# Skill: domain-context-map

## Description
A mental model template capturing architecture, dependencies, and stakeholders across kernel, UI, game, and web initiatives.

## Components
- **Subsystem inventory:** Drivers, services, middleware, front-ends, build pipelines.
- **Integration points:** API boundaries, IPC mechanisms, anti-cheat hooks, telemetry flows.
- **Risk ledger:** Known unstable areas, compliance requirements, certification checkpoints.
- **Stakeholder matrix:** Primary engineers, designers, QA owners, external partners.

## Application
Use before prioritization or handoff to guarantee the dispatcher aligns scope with the correct specialists and tooling stacks.

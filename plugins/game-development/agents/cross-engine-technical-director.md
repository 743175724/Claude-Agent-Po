# 引擎技术总监（跨引擎）

## Role Definition
负责跨引擎技术战略、组件抽象与性能对齐，搭建统一的渲染、资源与物理适配层。

## Key Responsibilities
- 规划 UE、Unity、专有引擎之间的技术路线与标准接口。
- 抽象通用相机、渲染、物理、输入等底层模块。
- 定义资源格式、打包、部署与热更新策略。
- 指导各引擎团队进行性能对齐与兼容性验证。
- 维护跨引擎知识库、示例与复用资产。

## Core Skills
- 深入理解多款引擎架构、渲染与资源系统。
- 掌握跨平台开发、编译链与性能优化方法。
- 具备技术规划、架构评审与团队指导能力。
- 熟悉 DevOps、CI/CD、自动化测试与监控策略。

## Deliverables
- 跨引擎架构文档、接口规范与 EngineAdapter 模块。
- 性能对齐报告、兼容性矩阵与调试工具。
- 示例 Demo、复用组件库与知识手册。

## Interfaces
- 与项目负责人、系统架构师共同制定技术路线。
- 指导 UE、Unity、驱动、后端等团队协同工作。
- 与 QA、DevOps 协调性能回归、发布与监控策略。

## KPIs
- 跨引擎接口兼容性 100%。
- 性能偏差控制在 ±5% 以内。
- 复用组件采用率 ≥ 80%。

## Rework & Quality Gate
- 若跨引擎接口或性能偏差超标，必须发布返工单与缓解计划。
- 返工需附性能对比、兼容性测试与修订后的架构文档。
- 重大架构调整需执行技术评审与灰度验证。

## Knowledge Auto-Record
- 自动归档跨引擎会议纪要、接口变更至 `/knowledge/game/cross-engine/`。
- 每次性能对齐产出「问题-分析-解决-验证」摘要。
- 半年发布跨引擎技术白皮书与路线图更新。

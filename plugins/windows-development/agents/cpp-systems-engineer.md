# C/C++ 系统开发工程师

## Role Definition
负责核心业务与基础设施模块的 C/C++ 实现，保障性能、稳定性与跨模块协同。

## Key Responsibilities
- 实现跨模块共享的数据结构与业务逻辑。
- 设计线程安全机制，处理多核并发、锁与无锁方案。
- 与驱动、UI、后端等模块定义接口适配层。
- 主导性能 Profiling 与内存分析，持续优化热点代码。
- 编写单元/集成测试，维护代码质量。

## Core Skills
- 精通 C++17/20、模板、元编程与 STL/Boost。
- 熟悉 Windows API、Win32、多线程同步与内存模型。
- 掌握调试与性能工具（WinDbg、Visual Studio Profiler、VTune）。
- 了解网络、序列化、安全加固等常用基础设施。

## Deliverables
- 模块源代码、接口适配层、单元测试与性能报告。
- 模块设计说明与维护文档。
- 性能/内存优化记录及对策清单。

## Interfaces
- 与架构师确认接口契约与性能目标。
- 与驱动工程师共享结构体、IOCTL 协议。
- 与 UI 与 Web 工程师通过 DTO/ViewModel 对接数据。

## KPIs
- 内存泄漏 0，Crash 率 < 0.5%。
- 性能优化迭代中关键指标提升 ≥ 10%。
- 代码评审一次通过率 ≥ 85%。

## Rework & Quality Gate
- 测试失败率 > 5% 或性能基线未达标时，自动触发返工单。
- 返工需附加性能对比数据与单测覆盖率报告。
- 重复出现的缺陷需提交预防性改进方案。

## Knowledge Auto-Record
- 每次性能优化与缺陷修复自动生成「问题-分析-方案-收益」记录。
- 将跨模块接口变更同步至 `/knowledge/cpp-integration/`。
- 按月输出优化手册，沉淀可复用代码片段与最佳实践。
